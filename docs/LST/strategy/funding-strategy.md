# Live Stream Translator – 펀딩 전략 문서

이 문서는 **프로젝트 펀딩 전략, 위험성 분석, 플랫폼별 비교, 실행 계획**을 정리한 문서입니다.

---

## 📊 1. 펀딩 플랫폼 비교 분석

### 1-1. 플랫폼별 특징

| 플랫폼 | 장점 | 단점 | 적합도 | 수수료 |
|--------|------|------|--------|--------|
| **Kickstarter** | 대중적, 높은 노출 | All-or-Nothing, 배송 의무 | ⚠️ 중간 | 5% + 결제 3~5% |
| **Indiegogo** | 유연한 펀딩, 진행형 프로젝트 OK | 수수료 높음 | ✅ 적합 | 5% + 결제 3~5% |
| **Patreon** | 지속적 수익, 압박 적음 | 초기 자금 모으기 어려움 | ✅ **최적** | 5~12% |
| **GitHub Sponsors** | 개발자 친화적, 수수료 0% | 금액 작음 | ✅ **강력 추천** | 0% |
| **Open Collective** | 투명성, 오픈소스 친화 | 덜 알려짐 | ✅ 적합 | 10% |
| **Ko-fi** | 간편, 수수료 낮음 | 펀딩 기능 약함 | ⚠️ 보조용 | 0~5% |

## ⚠️ 2. 위험성 분석

### 2-1. 법적/윤리적 위험

#### 🚨 AI 사용 관련 위험

**문제 상황:**
```
"이 개발자는 ChatGPT/Claude로 코드 짜서 
우리 돈 받아먹었어요!" 
→ 후원자 반발
```

**대응 방안:**

**1. 100% 투명하게 공개**
- "AI 보조 도구를 활용하여 개발합니다"
- GitHub README에 명시
- 펀딩 페이지에 명시
- 개발 블로그에서 과정 공개

**2. AI는 도구일 뿐임을 강조**
- "건축가가 CAD 쓰는 것처럼"
- "작곡가가 MIDI 쓰는 것처럼"
- "디자이너가 Photoshop 쓰는 것처럼"
- 최종 설계/판단/테스트는 개발자

**3. 실제 가치 증명**
- MVP 먼저 공개
- "AI 덕분에 더 빠르게 더 좋은 제품을"
- 코드 품질 > 코드 작성 방법
- 오픈소스로 모든 코드 공개

### 2-2. 배송/완성 의무 위험

#### 🚨 Kickstarter/Indiegogo 문제

**현실적 통계:**
- Kickstarter 프로젝트 35%가 늦게 완성
- 9%는 아예 완성 못 함
- 소프트웨어 프로젝트는 더 위험

**법적 문제:**
```
펀딩 받음 → 개발 실패 → 환불 요구/소송
```

**대응 방안:**

**1. All-or-Nothing 펀딩은 피할 것**
- 목표 달성 못 하면 아무것도 못 받음
- 압박과 스트레스 증가

**2. 단계별 목표 설정**
```
$5K: Chrome Extension MVP
$10K: 다중 플랫폼 지원
$20K: Desktop App
$50K: 전담 개발 (3개월)
```

**3. 달성 가능한 낮은 목표**
- 야심찬 $50K보다 현실적인 $5K
- 초과 달성이 실패보다 낫다

**4. 명확한 환불/취소 정책**
- Patreon: 언제든 취소 가능
- GitHub Sponsors: 압박 없음

### 2-3. 압박감 & 번아웃 위험

#### 🚨 1인 개발자의 함정

**위험 시나리오:**
```
펀딩 성공 → 기대치 상승 → 압박 → 번아웃 → 프로젝트 중단
→ 후원자 반발 → 평판 손상
```

**특히 위험한 상황:**
- 풀타임 직장 + 사이드 프로젝트
- 과도한 목표 금액 설정
- 비현실적 일정 약속
- 혼자서 모든 것 처리

**대응 방안:**

**1. 현실적인 일정 공개**
```markdown
⚠️ 먼저 알아주세요
- 저는 주중에 풀타임 직장을 다닙니다
- 개발 시간: 주중 2시간 + 주말 8시간 = 주 18시간
- 예상치 못한 일이 생길 수 있습니다
- 일정이 밀릴 수 있습니다 (그래도 완성합니다!)
```

**2. 버퍼 포함 일정**
- 3개월 걸릴 것 같으면 6개월 약속
- "조기 완성"이 "지연"보다 훨씬 좋음

**3. 압박 없는 플랫폼 선택**
- GitHub Sponsors / Patreon 권장
- Kickstarter는 피하기

**4. 커뮤니티 협력**
- 번역: 커뮤니티
- 디자인: 외주 가능
- 테스트: 베타 테스터

### 2-4. 커뮤니티 반발 위험

#### 🚨 오픈소스 + 펀딩 논란

**예상 반응:**
```
"오픈소스인데 왜 돈 받아요?"
"광고도 있고 구독도 있는데 펀딩까지?"
"무료라며? 돈 받으려고 만든 거 아니냐"
```

**대응 방안:**

**1. 명확한 포지셔닝**
```
펀딩 = 초기 개발 가속 (일회성)
구독 = 지속 운영 (선택)
광고 = 서버 비용 (무료 사용자)
```

**2. 재정 투명성**
```markdown
## 💰 펀딩 사용처

목표: $5,000

- 개발 시간: $3,000 (200시간 × $15/시간)
- 서버 비용: $500 (6개월)
- API 비용: $500 (테스트용)
- 디자인: $500
- 마케팅: $300
- 예비비: $200

모든 지출은 투명하게 공개합니다.
```

**3. 오픈소스 유지 약속**
```markdown
## 🔓 오픈소스 약속

✅ 펀딩 성공 여부와 관계없이 오픈소스 유지
✅ 모든 코드는 MIT 라이선스
✅ 커뮤니티 기여 환영
✅ 포크/수정 자유
```

## ✅ 3. 추천 전략: 하이브리드 접근

### 3-1. Phase 1: GitHub Sponsors (지금 당장) ⭐⭐⭐⭐⭐

#### 왜 GitHub Sponsors인가?

**장점:**
- ✅ 수수료 0% (GitHub가 부담)
- ✅ 개발자 커뮤니티 친화적
- ✅ 펀딩이 아닌 "후원" 느낌
- ✅ 압박 없음, 언제든 시작/중단 가능
- ✅ 오픈소스 프로젝트에 최적
- ✅ 지속적 수익 가능

**단점:**
- ⚠️ 대중적 인지도 낮음
- ⚠️ 비개발자 접근성 낮음
- ⚠️ 초기 금액 작음

#### 티어 구조 예시

```markdown
## GitHub Sponsors Tiers

### $5/월 - ☕ 커피 한 잔
- 후원자 명단에 이름
- 개발 진행 상황 월간 업데이트
- Discord 후원자 채널 접근

### $15/월 - 🍕 피자 한 판
- 위 혜택 +
- 투표권 (다음 기능 우선순위)
- 월간 Q&A 세션 참여

### $50/월 - 🎮 진지한 후원자
- 위 혜택 +
- Pro 기능 평생 무료
- 1:1 피드백 세션 (월 1회)
- 베타 기능 우선 체험

### $200/월 - 🏢 기업/스폰서
- 위 혜택 +
- 웹사이트 로고 노출
- 우선 기능 요청
- 기술 지원 우선
```

#### 예상 성과

**보수적 시나리오:**
- 1개월: 5명 × $10 = $50/월
- 3개월: 20명 × $15 = $300/월
- 6개월: 50명 × $20 = $1,000/월

**현실적 시나리오:**
- 1개월: 15명 × $12 = $180/월
- 3개월: 60명 × $18 = $1,080/월
- 6개월: 200명 × $25 = $5,000/월

**낙관적 시나리오:**
- 1개월: 30명 × $15 = $450/월
- 3개월: 150명 × $25 = $3,750/월
- 6개월: 500명 × $30 = $15,000/월

### 3-2. Phase 2: Patreon 병행 (3개월 후)

#### GitHub Sponsors와 차이

**Patreon 장점:**
- 더 대중적 (비개발자도 참여)
- 한국어 사용자 친화적
- 결제 수단 다양 (카드, PayPal)
- 커뮤니티 기능 강력

**전략:**
- GitHub Sponsors와 동일 티어
- "편한 플랫폼 선택하세요" 메시지
- 중복 후원 방지 안내

#### 티어 구조 (GitHub Sponsors와 동일)

```markdown
## Patreon Tiers

동일한 혜택을 제공합니다.
GitHub Sponsors 또는 Patreon 중 편한 곳을 선택하세요.

⚠️ 두 곳 모두 후원하실 필요는 없습니다!
```

### 3-3. Phase 3: Open Collective (6개월 후, 선택)

#### 언제 고려할까?

**적합한 시점:**
- 프로젝트가 어느 정도 성숙
- 커뮤니티 신뢰 구축 완료
- 사용자 10만+ 달성

**장점:**
- 완전 투명한 재정 공개
- 비영리 느낌
- 세금 혜택 (일부 국가)
- 단체/기업 후원 받기 좋음

**전략:**
- "더 투명한 재정 관리를 위해 Open Collective 추가"
- GitHub Sponsors / Patreon 병행 유지

### 3-4. ❌ 피해야 할 것: Kickstarter/Indiegogo (초기에는)

#### 왜 피해야 하나?

**위험 요소:**
1. All-or-Nothing 압박
2. 배송 의무 (소프트웨어도 "완성" 의무)
3. 실패 시 평판 타격
4. AI 사용 논란 가능성
5. 높은 기대치
6. 환불 문제

#### 언제 고려할까?

**적합한 시점:**
- ✅ MVP 완성 후
- ✅ 사용자 1만+ 확보 후
- ✅ Desktop App 같은 큰 프로젝트용
- ✅ "펀딩"이 아닌 "사전 판매" 개념
- ✅ 팀 확장/풀타임 개발 전환 시

**전략:**
- 초기 단계: GitHub Sponsors
- 성숙 단계: Kickstarter (큰 프로젝트)

## 📝 4. 펀딩 페이지 작성 전략

### 4-1. 투명성 & 정직함 (핵심!)

#### AI 사용 공개 예시

```markdown
## 🛠️ 개발 방식에 대하여

이 프로젝트는 AI 보조 도구(Claude, ChatGPT)를 
적극 활용하여 개발됩니다.

### 왜 밝히나요?
- 투명성이 신뢰의 기본이기 때문입니다
- AI는 "코드 생성기"가 아닌 "생산성 도구"입니다
- 건축가가 CAD를 쓰듯, 개발자가 AI를 씁니다

### 실제 개발 과정
1. 기능 설계 & 아키텍처 → 개발자 (나)
2. 보일러플레이트 코드 → AI 보조
3. 로직 구현 & 디버깅 → 개발자 + AI
4. 테스트 & 최적화 → 개발자
5. 최종 품질 책임 → 100% 개발자

### 이것의 의미
✅ 더 빠른 개발 (6개월 → 3개월)
✅ 더 많은 기능
✅ 더 나은 문서화
✅ 최신 기술 스택
❌ 품질 타협 없음
❌ 책임 회피 없음

### 여러분의 후원은...
"AI에게 주는 돈"이 아니라
"더 나은 제품을 만드는 개발자의 시간"에 대한 
투자입니다.

모든 코드는 오픈소스로 공개되며,
누구나 검증할 수 있습니다.
```

### 4-2. 현실적인 목표 & 일정

```markdown
## 📅 개발 로드맵 (솔직 버전)

### ⚠️ 먼저 알아주세요
- 저는 주중에 풀타임 직장을 다닙니다
- 개발 시간: 주중 2시간 + 주말 8시간 = 주 18시간
- 예상치 못한 일이 생길 수 있습니다
- 일정이 밀릴 수 있습니다 (그래도 완성합니다!)

### Phase 1 (지금~3개월)
✅ Chrome Extension MVP (완료!)
🔄 YouTube 자막 기능 (진행 중)
📅 Twitch 지원 (2월 목표, 버퍼 +1개월)

### Phase 2 (4~6개월)
- 커뮤니티 자막 플랫폼 베타
- SOOP/치지직 지원
- AI 자동 싱크

### Phase 3 (7~12개월)
- Desktop App
- 고급 기능들

### 💡 일정 관리 원칙
- 보수적으로 약속, 빠르게 달성
- 매주 진행 상황 업데이트
- 문제 발생 시 즉시 공유
- 투명한 소통

* 일정은 변경될 수 있으며, 항상 투명하게 공유합니다
```

### 4-3. 위험 요소 명시

```markdown
## ⚠️ 알려진 위험 & 제한사항

### 기술적 위험
- YouTube API 정책 변경 가능성
- 일부 플랫폼의 제약
- 브라우저 권한 문제
- 번역 API 비용 증가 가능성

### 개인적 제약
- 1인 개발자 (병가/긴급상황 가능)
- 풀타임 직장 병행
- 완벽주의 성향 (일정 지연 가능)
- 경험 부족 (첫 대규모 프로젝트)

### 대응 방안
- 오픈소스 (다른 개발자 참여 가능)
- 커뮤니티 피드백 기반 우선순위
- 최소 기능부터 출시 (MVP)
- 단계별 개발 (all-in 방지)

### 약속
❌ 완벽한 제품을 약속하지 않습니다
✅ 최선을 다한 제품을 약속합니다
✅ 투명한 소통을 약속합니다
✅ 중단하더라도 오픈소스 유지를 약속합니다
✅ 후원자 의견을 최우선으로 반영합니다
```

## 💡 5. GitHub Sponsors 설정 가이드

### 5-1. 프로필 작성 예시

```markdown
# 👋 안녕하세요!

저는 Live Stream Translator를 만들고 있는 개발자입니다.

## 🎯 이 프로젝트를 왜 만드나요?
```

## ✅ 8. DO & DON'T

### ✅ DO (해야 할 것)

1. **GitHub Sponsors 지금 시작**
   - 가장 안전하고 압박 없음
   - MVP 있으니 충분히 가능

2. **100% 투명성**
   - AI 사용 명시
   - 재정 공개
   - 일정 솔직하게

3. **낮은 목표, 높은 달성**
   - 첫 목표: $500/월
   - 과대 포장 금지

4. **커뮤니티 우선**
   - 후원자 = 파트너
   - 투표/피드백 참여
   - 감사 표현

5. **오픈소스 유지**
   - 펀딩 실패해도 프로젝트 계속
   - 코드는 항상 공개

6. **정기 업데이트**
   - 주간 진행 상황
   - 문제도 솔직하게 공유

### ❌ DON'T (피해야 할 것)

1. **Kickstarter 초기 런칭**
   - 위험 너무 큼
   - 나중에 Desktop App용으로

2. **AI 사용 숨기기**
   - 들키면 신뢰 붕괴
   - 당당하게 공개

3. **과대 약속**
   - "3개월 완성" → 실제 6개월
   - "완벽한 제품" → 버그 있음

4. **후원 = 의무 느끼기**
   - 번아웃의 지름길
   - 여유 있게 접근

5. **광고 + 구독 + 펀딩 동시에**
   - 욕심내지 말고
   - 단계적으로

6. **업데이트 중단**
   - 후원자와의 약속 깨짐
   - 최소 월 1회 업데이트

## 💡 13. FAQ (예상 질문)

### Q1: AI로 만든 코드인데 왜 돈을 받나요?

A: AI는 도구일 뿐입니다. 건축가가 CAD를 사용하듯, 개발자가 AI를 사용합니다. 설계, 구조, 품질 관리는 모두 개발자가 담당하며, 모든 코드는 오픈소스로 검증 가능합니다.

### Q2: 완성을 보장할 수 있나요?

A: 100% 보장할 수는 없습니다. 하지만 이미 MVP가 작동 중이며, 오픈소스이기 때문에 만약 제가 중단하더라도 커뮤니티가 이어갈 수 있습니다.

### Q3: 왜 Kickstarter가 아닌가요?

A: Kickstarter는 All-or-Nothing이고 배송 의무가 있어 압박이 큽니다. GitHub Sponsors는 지속적 후원 방식으로 더 유연하고 안전합니다.

### Q4: 후원을 중단하면 어떻게 되나요?

A: 언제든 중단 가능하며, 그동안 받은 혜택은 유지됩니다. Pro 기능은 후원 기간 동안만 유효합니다 (평생 혜택 제외).

### Q5: 수익이 목표보다 많으면?

A: 투명하게 공개하고 다음 단계 개발에 투자합니다. 팀 확장, 더 많은 플랫폼 지원, 인프라 개선 등에 사용됩니다.

### Q6: 세금 처리는?

A: 각 국가마다 다릅니다. 수입이 증가하면 세무사와 상담하는 것을 권장합니다. GitHub Sponsors/Patreon에서 세금 정보를 제공합니다.

### Q7: 기업 후원도 가능한가요?

A: 물론입니다! $200/월 티어 또는 별도 협의를 통해 로고 노출, 우선 지원 등의 혜택을 제공합니다.

   - AI 사용 명시

   - 재정 공개

3. **현실적 목표**
   - 낮게 시작
   - 단계적 확장

4. **오픈소스 유지**
   - 펀딩과 무관하게 프로젝트 지속
   - 커뮤니티 중심

**❌ Kickstarter는 나중에:**
- MVP 완성 후
- 사용자 1만+ 후
- 큰 프로젝트용

### 14-2. 마지막 조언

**펀딩은 "보너스"지 "의무"가 아닙니다.**

- 프로젝트가 충분히 좋으면 펀딩 없어도 성공합니다
- 펀딩은 개발 속도를 높이는 촉진제일 뿐입니다
- 커뮤니티와 제품이 먼저, 펀딩은 그 다음입니다

**가장 중요한 것:**
1. 실제로 작동하는 제품
2. 사용자가 원하는 기능
3. 투명하고 정직한 소통
4. 꾸준한 개발과 업데이트
5. 커뮤니티와의 신뢰 관계

### 14-3. 시작 타이밍

**지금 시작해도 됩니다:**
- ✅ MVP가 있다
- ✅ GitHub에 코드가 공개되어 있다
- ✅ 사용자가 있다 (베타 테스터라도)
- ✅ 명확한 로드맵이 있다
- ✅ 투명하게 소통할 준비가 되어 있다

**조금 더 기다려야 합니다:**
- ❌ 아직 프로토타입 단계
- ❌ 혼자만 쓰는 도구
- ❌ 로드맵이 불명확
- ❌ 시간 투자 각오가 안 됨

### 14-4. 예상 시나리오 (현실적)

**3개월 후:**
- 후원자: 20~50명
