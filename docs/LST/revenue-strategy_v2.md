## 🔐 4. 보안 전략 (심플)

### 4-1. Google OAuth 단일 로그인

**로그인 방식:**
```
[🔵 Google로 시작하기]
  ↓
Google 인증
  ↓
이메일, 이름, 프로필 사진 자동
  ↓
자동 로그인 완료
```

**왜 Google만?**
- 99% 사용자가 이미 보유
- YouTube = Google 계정 (타겟층 일치)
- 구현 단순, 신뢰도 높음
- 비밀번호 관리 불필요

### 4-2. 1인 1계정 강제

**DB 제약:**
```sql
users 테이블:
- user_id (PK)
- google_id (UNIQUE) ← 핵심!
- email (UNIQUE)
- created_at
- tier (free/pro/team)

제약:
- google_id UNIQUE
- 동일 Google 계정 재가입 불가
- 탈퇴 후 재가입 시 이력 보존
```

### 4-3. 리딤코드 중복 방지

**DB 구조:**
```sql
redemptions 테이블:
- user_id
- code
- redeemed_at

제약: UNIQUE(user_id, code)
→ 동일 유저가 같은 코드 재사용 불가
```

### 4-4. VPN 악용 대응

**방어 메커니즘:**
1. Google 계정 만들기 어려움 (인증 필요)
2. 리딤코드는 수동 배포 (랜덤 없음)
3. AI 패턴 분석 (의심 계정 감지)

**추가 방어 (선택):**
- 비정상 패턴 감지:
  - 가입 후 5분 내 Pro 전환
  - IP 지역 계속 변경
  - 동일 기기에서 여러 계정
- 의심 점수 시스템
- 수동 검토 → 차단

**결론:**
- 휴대폰 인증 불필요 (해외 사용자 진입장벽)
- Google OAuth만으로 충분한 방어
